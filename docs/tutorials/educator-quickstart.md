# Educator quickstart

Stand up a class, share its join code, assign a case and read the roster in
about ten minutes. This is the shortest happy path; each step links to the
full educator guide.

You see the role as **Teacher** in the UI; in the API and generated
reference it is `educator`, and a class is called a *cohort* — they are the
same thing. See the [Glossary](/reference/glossary).

## Before you start

Sign in with an account that has the **educator** role. You manage only
classes you own.

## 1. Create a class

1. Open **Settings → Classes**.
2. Type a name in **New class name** and click **Create**.

That is the whole minimum — a named class with no members or join code yet.
To add a description, dates, cases and co-teachers in one step, expand
**Add details, cases, co-teachers & students** before creating. Full
detail: [Classes & join codes → Create a class](/educator/cohorts).

## 2. Generate and share the join code

1. Open the class and go to the **Manage** tab.
2. Under **Join code**, click **Generate join code**.
3. Click **Copy** and share the code with your students.

Students enter it under **My Profile → Join a class**. Rotate or disable
the code if it leaks. Full detail:
[Classes & join codes → Join codes](/educator/cohorts).

::: warning
Anyone with the code can join the class. Rotate or disable it if it leaks.
:::

## 3. Assign a case

1. In the open class, go to the **Settings** tab.
2. Under **Assigned cases**, click **Add cases**.
3. Pick one or more cases from the library picker and click **Assign**.

Assignment records which scenarios belong to the class; it does not restrict
what students can run or change scoring. Full detail and what assignment
does/does not do: [Assigning cases](/educator/assigning-cases). For ideas on
which scenarios to assign, see the
[acute-cases walkthrough](/tutorials/walkthrough-acute-cases).

## 4. View the roster and a report

1. Open the class and go to the **Reports** tab (or use a report shortcut
   on the class card).
2. Read the **Roster** — per-student sessions, attempted, completed and last
   activity.
3. Click a student row to drill into their sessions.

A case counts as **completed** when the student reached the debrief. The
five reporting views (Roster, Completion grid, Analytics, Export, Live
feed): [Reporting & analytics](/educator/reporting).
