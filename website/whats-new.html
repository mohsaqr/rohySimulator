WHAT'S NEW SINCE V1.0.0

From virtual patient to clinical simulation platform.

Version 1 was a strong starting point: text chat, basic monitor, single-room workflow, session persistence, admin case editing, and multi-tenant auth. The current 2.5.1 line is substantially broader: multimodal patient interaction, multi-agent care teams, course-aware teaching operations, Oyon research capture, learning analytics, and enterprise-grade access control.

  • Case-bound system prompts
  • Basic monitor
  • Single-room layout
  • Session persistence

SIMULATION PLATFORM

The clinical encounter is now multimodal and multi-agent.

Rohy moved from a patient chat surface into a structured clinical environment with voice, avatars, rooms, investigations, treatments, and debrief.

Voice, avatars, and lipsync

Students can interact by text or voice with animated 3D patient avatars.

  • Google, OpenAI, Kokoro, and Piper TTS support.
  • Local-first TTS options for institutional deployments.
  • Viseme-driven lipsync and voice-mode subtitles.
  • Per-case and per-persona voice configuration.

Multi-agent care team

The learner now encounters a team, not only a chatbot.

  • Patient, nurse, consultant, family, and debrief tutor personas.
  • Page/call flows with server-anchored arrival timing.
  • Separate LLM routing per persona.
  • Knowledge scope controlled per agent and per case.

Five-room workflow

The single-room layout became a full clinical workspace.

  • Patient, examination, laboratory, radiology, and consultant rooms.
  • Room activity badges for unread or ready work.
  • Room-stamped learning events for later analysis.
  • Unified navigation state across the simulation.

Richer investigations

Labs and radiology now behave like first-class clinical tasks.

  • 225 labs across 33 groups.
  • 74 radiology studies.
  • Gender-specific reference ranges.
  • Realistic turnaround timing and report viewing.

Structured examination

The physical exam surface became more specific and teachable.

  • 67 named exam regions.
  • Anterior, posterior, and lateral anatomy support.
  • Inspection, palpation, percussion, and auscultation.
  • Special-test and body-map fixes for production use.

Live physiology and debrief

The monitor and post-case experience became more realistic.

  • Physiologic ECG generator and multi-parameter vitals.
  • Treatment engine with 33 default treatments.
  • Alarm routing through a central notification system.
  • Separate Socratic discussant for End & Debrief.
TEACHING OPERATIONS

Courses, users, enrollment, and assignment are now core product areas.

The platform now supports real teaching administration: rosters, registration codes, assignment, bulk operations, course reports, and controlled learner access.

Courses and cohorts

Teachers can manage classes instead of relying on informal case access.

  • Teacher-owned courses/classes.
  • Join and registration codes.
  • Co-teachers and course settings.
  • Per-course case assignment.

Roster and completion tools

Course-level reporting is built around the people actually enrolled.

  • Roster views.
  • Completion grid.
  • Live feed.
  • Course export.

Enterprise user operations

User administration is more scalable for larger deployments.

  • Bulk operations.
  • Bulk import workflow.
  • Better enrollment and assignment flows.
  • Improved alignment between users, courses, and cases.

Controlled student access

Students now see the right cases, not the entire catalogue.

  • Default course/default case remains available.
  • Assigned course cases are visible to enrolled students.
  • Unassigned cases are blocked from direct API access.
  • Unassigned session launch is blocked.

Settings Control Center

Administration now starts from a clearer command deck.

  • Card-based settings landing page.
  • Courses restored as a first-class settings area.
  • Oyon and Logs grouped under Analytics.
  • Existing settings workspaces preserved underneath.

Documentation and help

Operators and educators now have a real support surface.

  • VitePress documentation site.
  • Role-filtered in-app Help & Support drawer.
  • Release notes and diagnostics bundle.
  • Trainee, educator, admin, operator, integrator, and security guides.
RESEARCH AND ANALYTICS

Rohy now produces analyzable multimodal learning traces.

The analytics layer connects user activity, course context, rooms, cases, vitals, Oyon signals, logs, and learning sequences.

Oyon capture and analytics

Browser-side gaze and affect capture is integrated into the simulation.

  • MediaPipe and ONNX Web inference.
  • Aggregated 10-second windows.
  • Gaze AOI normalization.
  • Gaze maps by simulator room.

Patient eye contact

The avatar can now react to the learner's face position.

  • Patient and discussant track the learner's face.
  • Avatar glances at the monitor when alarms fire.
  • Eye contact resumes when the learner is still.
  • Oyon face stream powers the behavior.

Learning-event log

Simulation activity is encoded for research-grade analysis.

  • 130+ canonical verbs.
  • Room-stamped and vitals-enriched events.
  • Session activity feed.
  • Course-aware analytics and logs.

Course analytics

Teachers get useful course and student glimpses, not only global analytics.

  • Course filters in analytics and logs.
  • Student activity frequencies.
  • Course-level signals.
  • Case/session drill-downs.

Sequence and network analysis

The analytics stack supports research patterns directly.

  • Transition-network analytics.
  • Timeline and hourly views.
  • TNA sequence support.
  • Cluster and state-distribution summaries.

Data safety and visibility

Research data exposure is controlled more carefully.

  • Frozen-at-write Oyon visibility flags.
  • Tenant and ownership gates.
  • Centralized response redaction.
  • Role-aware access checks.

BOTTOM LINE

Version 1 was a simulator. The current line is a course-aware, research-ready clinical simulation platform.

The important shift is not one feature. It is the integration: cases, courses, users, assigned access, agents, rooms, vitals, voice, Oyon, logs, analytics, exports, and documentation now work as one platform surface.
